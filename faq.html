Websites

Website questions

What tools have been used to design the website?

To design the website, the Bootstrap web framework was used, as it provides fast deployment of a website with ready components, and prevents the user from having to design the website from scratch using Javascript and CSS from scratch. Bootstrap is open-source, and thousands of templates and resources are available to use.

Github was used to host the website, as Github Pages allow for hosting of a static website for free, while benefiting from Git version control

How to register a custom domain website?
A domain is registered by:
  1. Finding a domain name
  2. Picking a domain registrar such as GoDaddy, Amazon AWS or Google Domains
  3. Registering the selected domain
  4. Setting up the website on a web host such as Amazon AWS S3 (static), Github Pages, a custom HTTPD server, etc...
  5. Updating the DNS records using the registrar's service or a 3rd party service such as AWS Route 53
What makes a good website?
The following points are important to create a good website:
  1. Purpose: The website should have a purpose, or a concept behind it. It is likely that not many people will visit a website without a reason.
  2. Clarity: The main information should be easily findable and readable.
  3. Ease of navigation: The website should have a clear navigation method, with well defined pages and menus
  4. Speed: A website should load fast, and not use excessive resources
  5. Accessibility: Websites should ensure that impaired users are still able to obtain the required information in a straightforward manner.
How to build an accessible website?

Web Accessibility refers to "inclusive practice of ensuring there are no barriers that prevent interaction with, or access to, websites on the World Wide Web by people with disabilities" (Wikipedia) It is also defined as giving equal access to all users to information and functionality.

It is often categorised into different areas, with different mitigation approaches. The W3C WACG document issues guidelines to help designers create accessible websites. Guidelines include not relying on colours, better categorisation of on-screen elements, image replacement tags, etc.

How to implement GDPR?

General Data Protection Regulation is the EU 2016/679 regulation defining stricter privacy rules for entities processing user generated data, introduced in 2018. GDPR brings along a set of new rules to follow, requiring websites hosted in the EU to ensure data is handled correctly.

The following provisions are to be highlighted in GDPR:

  1. Consent: Users need to explicitely consent to their data being processed, and consent must be made clear and distinct from other enrollment. Users cannot be confused by legal texts anymore. Giving consent and revoking consent should be equal is ease.
  2. Breach notification: Data must be stored securely; however, in the event of a breach, users need to be notified within 72h of such an event.
  3. Right to access: Users need to be able to know what data is stored about them by the services. They must know what they consented for, the purpose for which the data is used.
  4. Right to be forgotten: Users must be able to revoke access to the processing of their data, as well as be able to request erasure of their personal data.
  5. Data portability: Users need to be able to request for a copy of all data stored about them
  6. Privacy by design: The default behaviour of GDPR-compliant systems should ensure privacy is respected at all times.

In the case of the Futility website, no data is stored; however, if forms with data submission were to be implemented, and cookies used, clear consent options should be given, and data stored on secure infrastructure.

Marketing the website
To market the website, several steps can be done:
  1. Online ad compaigns: ad services such as Google Adsense or Facebook Ads can be used to target potential customers
  2. SEO: The website can be search optimized by including keywords and keyphrases relevant to the product
  3. Kickstarter campaign: A Kickstarter campaign would bring publicity to the product, while providing the funds to start production
  4. Traditional media ad compaigns: Newspaper, TV and news coverage of the product can bring publicity.
